# 管理后台使用说明

##  概述

管理后台是专为系统管理员设计的独立管理界面，用于管理用户、活动、帖子、社团等内容。

##  登录方式

使用管理员账号登录：
- **用户名**：admin
- **密码**：123456

登录后会自动跳转到管理后台界面。

##  功能模块

### 1. 数据统计 (Dashboard)

**访问路径**：`/admin/dashboard`

**功能说明**：
- 显示系统核心数据统计
- 展示最近的活动和帖子
- 实时监控平台运营状况

**数据项**：
- 用户总数
- 活动总数
- 帖子总数
- 社团总数
- 最近活动列表
- 最近帖子列表

### 2. 用户管理

**访问路径**：`/admin/users`

**功能说明**：
- 查看所有注册用户信息
- 搜索用户（用户名、学号、状态）
- 编辑用户信息
- 启用/禁用用户账号

**操作说明**：
1. **查询用户**：输入搜索条件，点击"查询"按钮
2. **编辑用户**：点击"编辑"按钮，修改信息后保存
3. **禁用/启用**：点击状态按钮，确认后生效

### 3. 活动管理

**访问路径**：`/admin/activities`

**功能说明**：
- 查看平台所有活动
- 搜索活动（活动名称、分类）
- 删除违规活动
- 查看活动详情

**操作说明**：
1. **查询活动**：选择分类或输入关键词
2. **查看活动**：点击"查看"按钮
3. **删除活动**：点击"删除"按钮，确认后删除

### 4. 帖子审核

**访问路径**：`/admin/posts`

**功能说明**：
- 查看互助广场所有帖子
- 审核待发布的帖子
- 通过/拒绝帖子
- 查看帖子详情

**审核流程**：
1. 查看待审核帖子列表（状态为"待审核"）
2. 点击"通过"或"拒绝"按钮
3. 确认操作后帖子状态更新

**审核标准**：
- 内容是否合规
- 是否包含违规信息
- 是否符合发布规范

### 5. 社团管理

**访问路径**：`/admin/clubs`

**功能说明**：
- 查看所有注册社团
- 搜索社团（名称、分类）
- 删除违规社团
- 查看社团详情

**社团分类**：
- 学术类
- 体育类
- 艺术类
- 公益类

### 6. 食堂管理

**访问路径**：`/admin/canteens`

**功能说明**：
- 查看食堂信息
- 编辑食堂信息
- 删除食堂
- 查看食堂评分

**管理内容**：
- 食堂名称
- 地址
- 营业时间
- 联系电话
- 评分状态

### 7. 系统设置

**访问路径**：`/admin/settings`

**功能说明**：
- 基础设置（系统名称、描述）
- 维护模式开关
- 通知设置（邮件、短信）
- 系统信息查看

**设置项**：
- 系统名称：修改平台显示名称
- 维护模式：开启后用户端显示维护提示
- 邮件通知：开启/关闭邮件通知功能
- 短信通知：开启/关闭短信通知功能
- 活动提醒：开启/关闭活动提醒功能

## 🎨 界面说明

### 左侧导航栏

管理后台采用左侧导航菜单，包含所有管理功能模块：
- 数据统计
- 用户管理
- 活动管理
- 帖子审核

- 食堂管理

## 📱 访问方式

### 方式一：直接访问

管理员登录后直接访问：
```
http://localhost:3000/admin/dashboard
```

### 方式二：从学生端进入

1. 使用admin账号登录学生端
2. 点击右上角用户名
3. 选择"管理后台"

## 🔐 权限说明

### 管理员权限

- 查看所有用户信息
- 管理所有平台内容
- 审核帖子和活动
- 修改系统设置
- 启用/禁用用户

### 学生权限

- 无法访问管理后台
- 登录学生端后看不到管理入口
- 强制访问管理页面会被拦截

## ⚠️ 注意事项

1. **密码安全**
   - 管理员密码为 123456（仅开发环境）
   - 上线前必须修改为强密码
   - 建议启用双因素认证

2. **数据操作**
   - 删除操作不可恢复，请谨慎操作
   - 建议操作前备份重要数据
   - 批量操作建议分批进行

3. **审核规范**
   - 严格按照平台规则审核内容
   - 拒绝帖子应注明拒绝原因
   - 定期检查审核记录

4. **系统维护**
   - 开启维护模式前通知用户
   - 维护完成后及时关闭模式
   - 定期备份数据库

## 🛠️ 开发说明

### 后端接口

管理后台API统一使用 `/api/admin` 前缀：
- `GET /api/admin/statistics` - 获取统计数据
- `GET /api/admin/users` - 用户列表
- `PUT /api/admin/users/{id}/status` - 更新用户状态
- `GET /api/admin/activities` - 活动列表
- `DELETE /api/admin/activities/{id}` - 删除活动
- `GET /api/admin/posts` - 帖子列表
- `PUT /api/admin/posts/{id}/approve` - 审核通过
- `PUT /api/admin/posts/{id}/reject` - 拒绝帖子

### 前端路由

管理后台路由统一使用 `/admin` 前缀：
- `/admin/dashboard` - 数据统计
- `/admin/users` - 用户管理
- `/admin/activities` - 活动管理
- `/admin/posts` - 帖子审核
- `/admin/clubs` - 社团管理
- `/admin/canteens` - 食堂管理
- `/admin/settings` - 系统设置
